# Gadget Marketplace API

A production-ready, highly scalable asynchronous Python backend specifically designed to handle e-commerce traffic, strict security, and maintainable architecture.

## 🚀 Core Features & "How It Works"

### 1. High-Concurrency Asynchronous Foundation
- **The Tooling**: Built on entirely asynchronous Python utilizing **FastAPI** paired with an **uvicorn** server.
- **How It Works**: By leveraging Python's `async`/`await` event loops, the API accepts thousands of simultaneous network requests. It never blocks the main thread waiting for database queries to return, making it incredibly lightweight and lightning fast.

### 2. Database & Connection Multiplexing (PgBouncer)
- **The Tooling**: PostgreSQL, `asyncpg`, SQLModel, and **PgBouncer**.
- **How It Works**: 
  - Your schemas are strictly modeled using **SQLModel** and queried using the blazing-fast **asyncpg** driver.
  - *The Magic:* Postgres naturally consumes a lot of OS memory for every direct TCP connection. To prevent the database from crashing under stress, requests actually route to **PgBouncer** first. PgBouncer maps thousands of incoming FastAPI queries into a small pool of extremely efficient, pre-existing persistent connections with PostgreSQL running natively under SCRAM-SHA-256 encryption.

### 3. Automated Migrations (Alembic)
- **The Tooling**: Alembic Asynchronous branch.
- **How It Works**: Instead of manually scripting SQL tables, `migrations/env.py` has been configured to read your Pydantic environment configurations dynamically. Running `alembic revision --autogenerate` automatically scans your python SQLModels and natively writes `.py` scripts to safely update the database schema across any environment without losing data.

### 4. Scalable Service Layers (`services/`)
- **The Tooling**: Service Repository Pattern.
- **How It Works**: Rather than dumping complex business logic inside the FastAPI `routers/`, interactions with external utilities and databases happen precisely inside `services/auth_services.py`. This keeps endpoints razor-thin (making them incredibly easy to read and HTTP test) while rendering the business logic independently unit-testable.

### 5. Rate Limited Caching (Redis)
- **The Tooling**: Redis 7 and `fastapi-limiter`.
- **How It Works**: To prevent automated bot attacks and brute-force guessing of user passwords, a Redis cache runs alongside the database in Docker. When Uvicorn boots, the lifespan hook mounts Redis into FastAPI. Endpoints like `/login` are then decorated with a strict limitation (`5 times every 60 seconds`). If a single user exceeds this velocity, the server instantly shuts them down with a `429 Too Many Requests` mathematically verified by Redis.

### 6. Stateless Authentication
- **The Tooling**: `PyJWT` and `passlib` (bcrypt).
- **How It Works**: Using enterprise-grade `bcrypt` hashing, plaintext passwords are mathematically obscured before permanently residing in Postgres. Upon successful validation, the backend mathematically signs a JSON Web Token (JWT) strictly using the hidden `SECRET_KEY`. This is entirely **stateless**: the servers don't need a heavy central session table to look up who you are; they can cryptographically trust the token itself.

### 7. Deployment-Ready Architectures
- **The Tooling**: `docker-compose.prod.yml`, `Dockerfile`, `.gitignore`, and `.env`.
- **How It Works**: Following true "12-Factor App" methodologies, every strict or sensitive detail is confined exclusively to a `.env` file successfully ignored by `git`. When you are ready to go live, the entire 4-piece architecture (API container, DB container, Redis container, and PgBouncer container) are seamlessly generated by the `docker-compose.prod.yml` orchestrator completely identically to your local development array.

## 💻 Environment Setup Cheat Sheet

**Run Database & Utilities Locally:**
```bash
docker compose up -d
```

**Apply Required Database Tables (using Alembic):**
```bash
export PYTHONPATH=$PWD/..
alembic upgrade head
```

**Run the FastApi Server Locally:**
```bash
export PYTHONPATH=$PWD/..
uvicorn server.main:app --reload --port 8000
```
